```
Definition of ready:

* Issue heeft wireframe (als mogelijk)
* Userstory en acceptatiecriteria moeten aanwezig zijn voor elke issue (als ... wil ik ... zodat)
* Iemand aan de issue gekoppeld
* Weight aan de issue gekoppeld
* Issue is in de juiste kolom (To Do, In Progress, Review, Done)
* Issue heeft de juiste labels
```

```
Definition of done:

* Overeenkomst tussen de groep over de code
* Getest door de developer
* Gereviewed door PO
* Voldoet aan minstens 80% van de acceptatiecriteria
* Alle code is geformat met het `./vendor/bin/pint` commando

```

```
Code Conventions:

- Probeer je zo veel mogelijk aan de [PSR-12](https://www.php-fig.org/psr/psr-12/) coding style guide te houden.
- Zorg dat elk zelfgeschreven functie een comment heeft over wat het doet.

```
